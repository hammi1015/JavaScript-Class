1. var의 단점{: 사용자(개발자)의 의도와 상관없이 전역변수로 선언됨.
console.log(a)
var a = 10;
은 먼저 선언이 안되어도 가능
let이나 const는 선언이 먼저 되지 않으면 불가능
}
2. 중복 선언이 가능하다.{
var a = 10;
var a = 20;
console.log(a); == 20
소규모에서만 좋아보임, 대규모는 무쓸모
}

let은 재선언은 안되지만 재할당은 가능함
const는 재선언, 재할당이 안됌.


let, const를 써야 하는 이유
1. 코드 가독성
    const를 쓰면 재할당이 안된다라고 바로 알릴 수 있고, 중복선언방지 가능
    let을 쓰면 재할당은 된다고 알릴수 있고, 중복선언이 방지된다.
